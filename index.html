You hand it one task. It makes the agent finish.

A change enters easily but only reaches the centre, merged, by breaking through every ring: your tests, then a checker, then an independent reviewer. Each ring refuses half-finished work and sends it back. Chakravyuh runs the agent so you don't have to sit and watch it, and it never merges for you.

1Maker writes the code, sandboxed, TDD-first
2Your tests run. Red gate, no vote counts
3Checker reads it against the spec
3Reviewer reads it for quality, a different model
◉The centre committed to a branch. You merge

One attempt, ring by ring.

Every attempt runs in a throwaway git worktree with the maker's shell sandboxed. Fail any ring and the blockers feed back into the next try, up to three.

WORKTREE

Isolate

A fresh branch off your base, so a bad run touches nothing real.

MAKER

Build

Pi writes the slice with edit and write tools, confined to the worktree.

HEALTH GATE

Prove

Your command runs. Non-zero exit ends the attempt before any model votes.

CHECKER ∥ REVIEWER

Judge

Two read-only agents run at once: spec, and standards. Both must pass.

COMMIT

Land

Gate green and both approve, so the change commits on its own branch.

Why it looks like this.

Your tests are the boss

The checker and reviewer are LLMs, so they can be wrong. Your health command is a plain exit code, and it runs before any model vote counts.

The maker can't grade its own work

Enforced by tools, not trust. The maker gets edit and write; the reviewers get read-only, on a different model, so it's a genuine second look.

The limits come from outside

The loop bounds itself on attempts, an idle timeout, a hard timeout, and a token cap, never on dollars the provider may misreport.

A missing verdict counts as no

If a reviewer's verdict is missing or unparseable, that's a rejection. We never read silence as approval.

Isolation in layers

Each attempt runs in its own git worktree. After the maker finishes, the real repo has to be clean, and a leak guard fails the task if it wrote anywhere else. On top of that the maker's shell is boxed in by a sandbox, because a clean worktree can't stop a rogue command.

Try the loop harness in 60 seconds.

Standalone and offline. This proves the harness builds and its logic is sound, with the coding agent fully mocked.

A real run drives a Pi coding agent against your own repo. TypeScript, no framework, Apache-2.0.

# clone, build, and run the test suite
git clone https://github.com/maahaa-dev/chakravyuh
cd chakravyuh
npm install
npm run build   # tsc, must be clean
npm test        # 167 unit tests, agent fully mocked
